dh交换算法原理

发表于 2021-12-04 | 分类于 security ， dh

交换过程描述

甲乙双方按照dh算法交换密钥。
首先，甲生成随机数x，底数为d，一个大素数p，根据算法生成A = d ^ x % p。之后甲将d、p、A传递给乙。
乙收到后生成随机数y，计算B = d ^ y % p, C1 = A ^ y % p，将B传回给甲。
甲收到B后计算C2 = B ^ x % p。
C1和C2就是协商得到的密钥。可以证明C1=C2。

证明

按照上述过程描述，欲证C1 = C2，C1 = A ^ y % p = (d ^ x % p) ^ y % p，C2 = (d ^ y % p) ^ x % p。
假设d ^ x = pn + a，则
C1 = (d ^ x % p) ^ y % p = a ^ y % p = (d ^ x - pn) ^ y % p
根据二项式定理，得到C1 = d ^ (xy) % p，同理，C2 = d ^ (xy) % p = C1。

算法的可靠性

由交换过程可以知道，攻击者可以拿到d、p、A、B，要破解算法，关键要拿到x、y。以x为例，A = d ^ x % p，现在就是已知其它三个变量求x。已知在数学上，当p很大时，很难计算x（有限群的离散对数问题），这就保证了该算法的可靠性。
