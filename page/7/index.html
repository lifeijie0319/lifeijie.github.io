redis应用

发表于 2022-04-10 | 分类于 database ， redis

常见应用

  • 数据缓存
  • 分布式锁
  • 消息队列

缓存

缓存穿透

缓存穿透是指查询一个不存在的数据，查询没有结果不进行缓存，导致每次都要进行数据库查询。可以通过查不到数据设置很快过期的空值缓存来避免缓存穿透。

缓存与数据库的一致性

  • 合理设置过期时间
  • 写数据库时同步更新redis缓存，通过数据库事务机制保证一致性

分布式锁

由于redis是单线程，两个客户端顺序使用setnx(set if not exists) lock value，只有前一个会成功，成功后执行临界区代码，之后使用del删除lock，其它客户端可以获取锁。

问题与改进

  1. 客户端A设置的锁被客户端B删除
    value与客户端一一对应，删除的时候比较value，确认和加锁的时候是一个客户端
  2. 加锁后客户端宕机锁无法释放导致死锁
    设置锁的超时时间，超过一定时间自动释放锁。Redis 2.6.12 之后，Redis 扩展了 SET 命令的参数，可以使用set key value [EX seconds] [PX milliseconds] NX代替setnx
  3. 超时时间的决策
    Redisson框架使用锁续期的策略实现。思路是客户端加锁后新建守护线程，守护线程定时給锁续期，客户端正常释放锁后关闭守护线程。如果客户端宕机，守护线程随之退出，redis的锁在超时时间过后自动失效。
  4. 锁获取失败后重复获取
    使用发布订阅模式，获取锁失败后订阅锁释放消息，锁释放后发布释放消息。
  5. 锁重入
    使用Redis的哈希表存储可重入次数，当加锁成功后，使用hset命令，value(重入次数)则是1，如果同一个客户端再次加锁成功，则使用hincrby自增加一。解锁时，先判断可重复次数是否大于0，大于0则减一，否则删除键值，释放锁资源。
