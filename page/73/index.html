mybatis缓存

发表于 2022-04-10 | 分类于 language ， java ， database ， mybatis

一级缓存

基于 PerpetualCache 的 HashMap 本地缓存，其存储作用域为 Session。默认打开一级缓存。

一级缓存的命中原则

  • 同一个环境（数据库）
  • sql的id相同
  • sql的内容相同
  • sql的传参相同（主要是最终传入sql中的参数值要相同）
  • 分页参数相同

    生命周期

    缓存产生

    session执行select*方法时，即需要是查询方法。

    缓存结束（被清空）

  • session关闭
  • session提交或者回滚
  • 执行update/delete/insert
  • 执行session.clearCache

    spring与以及缓存

    如果未开启事务，每次查询会新建session，缓存不生效。开启事务的情况下，同一事务下的多次查询能使用以及缓存。

二级缓存

二级缓存与一级缓存其机制相同，不同在于其存储作用域为SessionFactory，并且可自定义存储源，如 Ehcache。默认不打开二级缓存。

配置打开二级缓存

  • 在mybatis-config.xml中配置
  • 在映射关系配置文件的mapper标签下增加
  • 让相关的实体类implements Serializable

二级缓存的命中原则

与一级缓存的命中原则一致。

生命周期

缓存产生

  1. session执行select*方法时，即需要是查询方法。
  2. session执行close或者commit。

    缓存结束（被清空）

    进行了C/U/D 操作并提交事务后，默认该mapper下所有 select 中的缓存将被 clear。

    其它常见配置

    useCache

    select标签的属性，规定查询是否使用缓存，默认true。

    flushCache

    作为select标签的属性时默认false，作为insert/update/delete标签的属性时默认true。表示是否清空二级缓存。

    eviction

    标签的属性。
    当二级缓存达到一定大小时，会自动进行清楚，可选清除策略如下：
  • LRU 移除时间最长不被使用的对象
  • FIFO 先进先出
  • SOFT 基于GC和软引用移除对象
  • WEAK 基于GC和弱引用移除对象

    size

    标签的属性。
    引用数目，当引用数目达到size时，按照eviction策略进行清除。
