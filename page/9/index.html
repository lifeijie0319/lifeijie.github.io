ulimit

发表于 2021-05-11 | 分类于 linux

定义

ulimt是一个linux自带应用工具，用于控制资源的可用性

使用

ulimit [-HSTabcdefilmnpqrstuvx [limit]]

-H 和 -S

-H选项和-S选项分别表示对给定资源的硬限制（hard limit）和软限制（soft limit）进行设置。硬限制（hard limit）一旦被设置以后就不能被非root用户修改，软限制（soft limit）可以增长达到硬限制（hard limit）。如果既没有指定-H选项也没有指定-S选项，那么硬限制（hard limit）和软限制（soft limit）都会被设置。limit的值可以是一个数值，也可以是一些特定的值，比如：hard，soft，unlimited，分别代表当前硬件限制、当前软件限制、不限制。

查看进程打开文件的最大限制

系统级最大限制

cat /proc/sys/fs/file-max

用户级限制

ulimit -n

查看进程已经打开的文件数

查看进程资源限制

cat /proc/$pid/limits

查看实际打开文件数

ls /proc/$pid/fd|wc -l

修改限制

临时修改

ulimit -n $num

永久修改

vi /etc/security/limits.conf
参考文档内说明配置限制项

实践

20210510 riak服务异常

现象

客户端连接riak服务端超时。检查发现进程打开的文件数已经超过用户设置的限制50000。另外top命令显示系统cpu资源占用率400%以上。

处理方案

修改riak用户的打开文件数限制，增加到65535。重启riak服务。

总结

riak为什么打开文件数这么多还有待研究。
