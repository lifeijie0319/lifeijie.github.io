sm2常见问题

发表于 2022-03-04 | 分类于 language ， java ， security

sm2加解密

在java中，一般使用bouncycastle库来实现sm2加解密，但是有几个要注意的点。

加解密排序模式

在sm2加解密过程中，密文由C1C2C3三个模块组成，但是密文的排序会有两种方式。按照国密标准，排序方式是C1C3C2，但是bouncycastle库模式使用C1C2C3排序。

密钥格式

  • 私钥 bouncycastle库使用32字节的私钥
  • 公钥 bouncycastle库使用64字节的公钥，但是在公钥前面会有额外一个字节的压缩标志位。在椭圆曲线算法中，公钥其实是曲线上的一个点，在bouncycastle库中用ECPoint来表示。它的二进制表示通过public byte[] getEncoded(boolean compressed)方法获得。这里如果compressed是true，那么压缩标志位会是02或者03，压缩的公钥只能在bouncycastle库中使用；如果是false，那么压缩标志位是04。

    密文

    使用bouncycastle库加密生成的密文会在前面自动加上一个额外字节04。

sm2加验签

在java中，使用bouncycastle库来实现sm2加验签，要注意的是，签名后会使用ASN.1的DER格式对签名结果（64字节）进行编码，处理后的结果可能是70-72字节的长度。
