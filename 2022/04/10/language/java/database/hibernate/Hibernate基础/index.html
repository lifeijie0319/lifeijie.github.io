ORM

ORM(Object Relational Mapping)，即对象-关系映射。面向对象的是编程中主流的开发方法，而关系型数据库是存储数据的主流系统。对象和关系数据是数据在内存和数据库中的不同表现形式，ORM是实现程序对象到数据库关系数据的映射转化关系的中间件。

实体类定义常见问题

实体类的成员变量使用基本类型还是包装类型

当成员变量是基本类型时，如果对应的数据库列存在null，那么获取数据映射到对象时会出现错误，因为基本类型不能存储null。

无参构造器

每个Hibernate实体类必须包含一个无参数的构造器， 这是因为 Hibernate框架要使用Reflection API，通过调用Class.newInstance()来创建这些实体类的实例。如果在实体类中找不到无参数的构造器，这个方法就会抛出一个InstantiationException异常。

对象状态与缓存

hibernate对象状态

  • transient 对象创建出来，没有与数据库建立联系
  • persistent 调用了save()/saveOrUpdate()等方法，数据持久化到数据库，生成id之后
  • detached session关闭之后

    一级缓存

    与Session绑定的缓存。
    同一个session中，只有第一次查询会实际执行sql，后面的查询会直接使用已经加载到内存的对象的数据。
    Session为清除缓存提供了clear(清除所有的一级缓存)、evict(清除实例对象缓存)、refresh（重新查询数据并刷新缓存）。
    一级缓存的数据存放在栈中。

    二级缓存

    与SessionFactory绑定的缓存。
    二级缓存默认不开启，需要特殊配置。
    二级缓存的数据存放在堆中。
