延迟加载

基本知识

Mybatis仅支持association和collection关联对象的延迟加载。association是一对一，collection是一对多。在Mybatis配置文件中可以配置是否开启延迟加载。相关的主要配置项是：

  • lazyLoadingEnabled 延迟加载的全局开关，默认false不开启。在特定的关联关系中可以设置 fetchType 属性来覆盖该项的开关状态。
  • aggressiveLazyLoading 当开启时，任何方法的调用都会加载该对象的所有属性。否则，每个属性会按需加载。默认false。

    延迟加载的原理

    创建目标对象的代理对象，如果获取延迟加载的对象发现是null，就发出预存的sql查询相关数据填充当前对象。

    延迟加载出现序列化问题

    使用延迟加载后，常会出现如下错误： debug观察此时通过dao获取到的结果，是一个代理对象，代理对象包含一个handler属性，该属性不是可以自动序列化的类型。此时，在返回结果进行json序列化的时候抛出异常。有两种常见的处理方式：
  1. 报错的信息建议disable SerializationFeature.FAIL_ON_EMPTY_BEANS，可以通过设置spring.jackson.serialization.FAIL_ON_EMPTY_BEANS=false来处理。背后的原因在于：
    FAIL_ON_EMPTY_BEANS是com.fasterxml.jackson.databind.SerializationFeature枚举类的一个枚举值，该值的作用是：Feature that determines what happens when no accessors are found for a type (and there are no annotations to indicate it is meant to be serialized). If enabled (default), an exception is thrown to indicate these as non-serializable types; if disabled, they are serialized as empty Objects, i.e. without any properties. 即FAIL_ON_EMPTY_BEANS=true，无法序列化属性时抛出异常，否则将属性值置空处理。
  2. 在被代理的实体类上添加注解@JsonIgnoreProperties(value = “handler”)。

    懒加载失效

    使用json序列化的时候，往往会调用方法获取关联数据，使得懒加载失效。（待验证）

插件

Mybatis自定义插件针对Mybatis四大对象进行拦截，分别是：

  • Executor(update, query, flushStatements, commit, rollback, getTransaction, close, isClosed)：拦截执行器的方法(log记录)
  • StatementHandler(prepare, parameterize, batch, update, query) ：拦截Sql语法构建的处理
  • ParameterHandler(getParameterObject, setParameters) ：拦截参数的处理
  • ResultSetHandler(handleResultSets, handleOutputParameters) ：拦截结果集的处理
