email传输的过程与协议

发表于 2021-11-02 | 分类于 network

过程理解

email传输是现实邮件传输的模拟。普通邮件传输，发送者写好信件，投入本地邮局的邮箱，邮局定时将邮件收走，按照目标地址投放到对应的目标邮局，目标邮局的邮递员根据实际目标地址投送到指定邮箱，最后接受者到邮箱取到信件。在email的世界里，我们用的邮件客户端（比如foxmail）就是邮箱，email服务提供商（比如网易、腾讯）就是邮局。

邮件传输过程中用到的协议

我们称邮件客户端为MUA(Mail User Agent)，email服务提供商为MTA(Mail Transfer Agent)。邮件的传输过程为：发件人 -> MUA -> MTA -> MTA -> 若干个MTA -> MDA <- MUA <- 收件人。这里MDA(Mail Delivery Agent)是接受邮件的服务商存放某些邮件的地方，可以理解为一个暂存服务器。
发邮件时，MUA和MTA使用的协议就是SMTP：Simple Mail Transfer Protocol，后面的MTA到另一个MTA也是用SMTP协议。
收邮件时，MUA和MDA使用的协议有两种：POP：Post Office Protocol，目前版本是3，俗称POP3；IMAP：Internet Message Access Protocol，目前版本是4，优点是不但能取邮件，还可以直接操作MDA上存储的邮件，比如从收件箱移到垃圾箱，等等。
