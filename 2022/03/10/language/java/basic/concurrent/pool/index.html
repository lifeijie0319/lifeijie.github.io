线程池

发表于 2022-03-10 | 分类于 language ， java ， basic ， concurrent

四种线程池

java提供四种线程池，分别为

  1. newSingleThreadExecutor 创建只有一个线程的线程池，当该线程正繁忙时，对于新任务会进入阻塞队列中(无界的阻塞队列)
  2. newFixedThreadPool 创建可容纳固定数量线程的池子，每个线程的存活时间是无限的，当池子满了就不再添加线程了；如果池中的所有线程均在繁忙状态，对于新任务会进入阻塞队列中(无界的阻塞队列)
  3. newScheduledThreadPool 创建一个固定大小的线程池，线程池内线程存活时间无限制，线程池可以支持定时及周期性任务执行，如果所有线程均处于繁忙状态，对于新任务会进入DelayedWorkQueue队列中，这是一种按照超时时间排序的队列结构
  4. newCachedThreadPool 当有新任务到来，放入队列，在池中寻找可用的线程执行任务，没有则创建线程执行，若空闲时间超过设置，线程自动销毁
    他们本质上都是创建ThreadPoolExecutor类。

ThreadPoolExecutor构造参数

  • int corePoolSize 核心线程数量
  • int maximumPoolSize 最大线程大小
  • long keepAliveTime 超过corePoolSize的线程保持时间
  • TimeUnit unit 时间单位
  • BlockingQueue workQueue 任务队列
  • ThreadFactory threadFactory 线程池工厂
  • RejectedExecutionHandler handler 拒绝策略

线程池任务执行流程

  1. 当线程小于corePoolSize时，新提交的任务将创建新线程执行，即使线程池中存在空闲线程。
  2. 当线程池中的线程数达到corePoolSize时，新提交的任务被放入workQueue中，等待线程池调度运行。
  3. 当workQueue已满，并且maximumPoolSize > corePoolSize时，新提交任务将创建新线程执行。
  4. 当提交任务数超过maximumPoolSize时，新提交的任务由RejectedExecutionHandler处理
  5. 当线程中超过corePoolSize数量的线程空闲时间超过keepAliveTime，释放空闲线程
  6. 当设置allowCoreThreadTimeOut=true时（默认false），线程池中corePoolSize范围内的线程达到keepAliveTime也将释放。

    内置线程池常见问题

    任务堆积队列过长内存不足问题

    newFixedThreadPool使用LinkedBlockingQueue来作为阻塞队列，该队列是没有上限的，所以如果任务过多，而线程池处理不过来，会导致该队列不断增大，撑爆内存。考虑使用ArrayBlockingQueue这种有界阻塞队列。newSingleThreadExecutor有同样的情况。newScheduledThreadPool使用DelayedWorkQueue，也是无界阻塞队列，有类似问题。

    无限制创建线程导致内存不足或者超过系统上限无法创建新线程

    newCachedThreadPool创建的线程池使用任务队列使用的是SynchronousQueue,SynchronousQueue本身并不存储任务，而是对任务直接进行转发，这本身是没有问题的，但maximumPoolSize被设置成了Integer.MAX_VALUE，所以由于 CachedThreadPool 并不限制线程的数量，当任务数量特别多的时候，就可能会导致创建非常多的线程，最终超过了操作系统的上限而无法创建新线程，或者导致内存不足。

线程池的关闭

线程池的状态

  1. 当创建线程池后，初始时，线程池处于RUNNING状态，此时线程池中的任务为0；
  2. 如果调用了shutdown()方法，则线程池处于SHUTDOWN状态；
  3. 如果调用了shutdownNow()方法，则线程池处于STOP状态；
  4. 当所有的任务已终止，线程池会变为TIDYING状态。接着会执行terminated()函数。
  5. 线程池处在TIDYING状态时，执行完terminated()之后，就会由 TIDYING -> TERMINATED，线程池被设置为TERMINATED状态。
