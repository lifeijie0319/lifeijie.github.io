Linux下的静态库与动态库

发表于 2021-12-04 | 分类于 language ， c

库的理解

本质上来说，库时一种可执行代码的二进制形式，可以被操作系统载入内存执行。一般分为静态库和动态库两种。

静态库和动态库的区别

命名

静态库文件名的命名方式是“libxxx.a”,库名前加”lib”，后缀用”.a”，“xxx”为静态库名。
动态库的命名方式与静态库类似，前缀相同，为“lib”，后缀变为“.so”。

链接方式

静态库的代码是在编译过程中被载入程序中，是将整个函数库的所有数据都整合进了目标代码。
动态库在编译的时候并没有被编译进目标代码，而是当你的程序执行到相关函数时才调用该函数库里的相应函数。

优缺点

静态库的优点是在编译后的执行程序不在需要外部的函数库支持，因为所使用的函数都已经被编进去了。缺点是，如果所使用的静态库发生更新改变，你的程序必须重新编译。
动态库的优点是动态库的改变并不影响你的程序，所以动态函数库升级比较方便。缺点是因为函数库并没有整合进程序，所以程序的运行环境必须提供相应的库

静态库的生成和使用

初始文件：time.c时间库函数源文件；time.h时间库函数头文件；test.c主入口文件。
步骤：
（1）编写库文件：time.c
（2）生成.o文件：gcc -c time.c
（3）生成静态库：ar cr libmytime.a time.o
注：c：表示创建静态库;r：表示将.o目标文件加入到静态库中；
（4）编译生成可执行文件：gcc -o test test.c -static -lmytime -L.
注：如果编译出现/usr/bin/ld: cannot find lc，可能的解决方法是sudo yum install glibc-static
（5）执行生成的可执行程序：./test

动态库的生成和使用

初始文件：同上；
（1）同上；
（2）生成.o文件：gcc -fPIC -c time.c
（3）生成动态库：gcc -shared -fPIC -o libmytime.so time.so
（4）编译生成可执行文件：gcc -o test2 test.c -lmytime -L.
注：可以看到，生成的可执行文件比4静态库生成的可执行文件小得多。
（5）执行生成的可执行程序：./test2
注：这里执行报错，因为动态库的查找路径一般不包括当前路径。可以在动态库默认搜索路径/usr/lib下添加链接文件，也可以将当前路径加入LD_LIBRARY_PATH环境变量。

三种使得装载器正确寻找并加载到动态库的方法

a. 将libx.so移到系统库文件目录下
b. export LD_LIBRARY_PATH=$LD_LIBRARY_PATH:`pwd`
c. 使用配置文件告诉系统动态库的路径. 
vi   /etc/ld.so.conf.d/xxxx.conf
