effective go 学习整理一

发表于 2022-03-04 | 分类于 language ， go ， effective_go

Formatting

gofmt 按照标准自动格式化源文件

Commentary

块注释除了基本的注释一段文字外，还常用于描述包信息。

godoc

godoc根据go源文件生成解释文档，顶层声明对象的注释信息影响生成的文档的质量。原文为：Comments that appear before top-level declarations, with no intervening newlines, are extracted along with the declaration to serve as explanatory text for the item.
每个都应该有包注释，包内有多个源文件时，只要在其中一个文件的顶部写包注释即可。
顶层声明对象的注释文档最好以对象名称开头。

Names

go命名一般风格是驼峰和帕斯卡

package names

包名一般是小写单个单词
By convention, packages are given lower case, single-word names; there should be no need for underscores or mixedCaps.

getter和setter

比如属性owner，按照go的风格，其getter方法名应为Owner()，setter方法名应为SetOwner()

interface names

单方法的接口命名一般是在方法后面加-er后缀，比如Reader接口实现了read方法

Semicolons

表示一行结束的分号不用加，go编译器会处理。所以不能把控制结构(if, for, switch, or select)的开始括号另起一行。

    Functions

    多返回值

    go函数支持多返回值，且返回值支持命名，一旦命名，这些命名的返回值就是本地变量，初始化为各自类型的零值，且return时无须再加上这些命名返回值。

    defer

    defer将一个函数的执行放在当前调用defer的函数结束前执行，常用于资源释放。
    传给defer函数的参数在defer这行出现时就执行。
    多个defer函数按照LIFO的顺序执行。
