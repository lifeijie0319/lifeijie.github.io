jsp与servlet的联系与区别

jsp本质上就是servlet，但是这是web容器编译后的情况，可以认为jsp是一种专门用来渲染页面的被简化的servlet，程序员只需要填入需要的数据，由容器完成后续编译成servlet的工作。
| | jsp | servlet |
| ——– | ————————————————————– | ——————————- |
| 作用 | 渲染页面 | 业务逻辑处理，提供jsp需要的数据 |
| 内置对象 | 通过HttpServletRequest/HttpServletResponse/HttpServlet对象得到 | 无 |

jsp的内置对象与作用

  • request 客户端请求对象
  • response 响应对象
  • pageContext 页面上下文，可以引用到request/session/application等作用域
  • session 用户会话对象
  • application 服务器运行环境对象
  • out 输出流对象
  • config web应用的配置对象
  • page jsp页面本身，相当于类中的this
  • exception 封装页面抛出的异常

jsp的四种作用域

  • page 页面相关的对象和属性
  • request 请求相关的对象和属性，一个web请求可以跨越多个page
  • session 用户与服务器建立的会话相关的对象和属性，跨越request
  • application 代表与整个web应用相关的对象和属性，一个全局作用域
