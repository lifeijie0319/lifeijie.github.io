主键相关问题

发表于 2021-04-04 | 分类于 database ， mysql ， index

为什么建议要建主键

因为MySQL底层使用B+树结构存储数据，主键索引是天然的构建依据。如果没有建立主键，mysql内部会建立隐藏主键。

主键为什么建议用自增整型数据

首先，整型便于比较，在B+树的数据操作中，数据比较是最常见的操作，整型数据在比较的时候比其他类型都快。而且一般来说，整型数据的存储空间较小。
自增又有什么好处？索引值自增时，插入的数据永远在B+树的最右侧子节点。如果不自增，插入的数据可能在树的中间，导致叶子节点页分裂（页已经被原有数据占满），有一个再平衡的过程，效率不高。所以自增索引减少了B+树的结构调整，提高了存储效率。
