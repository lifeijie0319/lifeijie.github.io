CSRF攻击

发表于 2022-03-04 | 分类于 security ， web

什么是CSRF

跨站脚本伪造(CROSS SITE REQUEST FORGE)，即CSRF，是指攻击者在自己的网站上伪造用户请求访问用户曾经访问的已登录认证的有CSRF漏洞的网站。由于被攻击网站只确认了请求来自于用户浏览器，而没有确认请求是否来自用户的真实意愿。

如何防范CSRF

验证HTTP referer字段

HTTP头中的Referer字段记录了该HTTP请求的来源地址。referer字段告诉服务器该网页是从哪个页面链接过来的，而攻击者的网页和被攻击网站的页面并不相同。

使用验证码

对重要的请求，比如转账等，添加额外校验。

token验证

CSRF能够成功的关键在于用户访问攻击者网站而在未知情的情况下向缺陷网站发送了请求，这个请求带上了用户之前访问缺陷网站登录验证的cookie。所以，在cookie之外向服务端传递之前服务端产生的token，服务端将请求的token和服务端存储的token（一般在session中）比较确认请求是否来自于自己的网站页面。token可以放在报文体或者http头部中。
