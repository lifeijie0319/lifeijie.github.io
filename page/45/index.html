jar包是什么

jar包是zip格式的压缩包，压缩包里面是按照目录组织的.class文件。

可执行的jar包是什么情况

如上所示是一种常见的执行java程序的命令，之所以可以这样执行，是因为a.jar中除了必须的组织好的.class文件，还指示了主类即程序入口，这个是由/META-INF/MANIFEST.MF文件的Main-Class配置值项指示的。
另外，jar包还可以包含其它jar包，这个时候，就需要在MANIFEST.MF文件里配置classpath了。

原始jar打包方式

说明：cvfe是最常见的选项，代表含义分析别如下：

  • -c create new archive
  • -v generate verbose output on standard output
  • -f specify archive file name
  • -e specify application entry point for stand-alone application bundled into an executable jar file
    c对应后面的[jar-file]，指要创建的jar文件名，f对应后面的[manifest-file]，指要手动配置MANIFEST.MF文件，e对应后面的[entry-point]，代表程序入口类。[-C dir]指示了打包前要切换的目标路径。最后files指示要打包的.class文件，如果这里是目录的话，会进行递归查找。
