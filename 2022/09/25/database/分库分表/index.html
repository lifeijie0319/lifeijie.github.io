分库分表

发表于 2022-09-25

为什么要分库分表

随着业务增长，数据量增大，关系数据库数据查询的效率下降。为了减轻数据库压力，提升数据库响应速度，可以考虑分库分表。分库分表本质在于数据的拆分，一般分为垂直分表、垂直分库、水平分表、水平分库。

垂直分表

垂直分表是按照表的列进行拆分，将不常用的列、字段占用空间大的列拆分出新的表。

为什么垂直分表能提高数据库查询效率

  1. 大字段本身占用了更多空间，读取相应的字段花费时间更长
  2. 页是数据存储的单位，大字段占用空间大，单页存储的行数少，IO效率低。
  3. 数据库以行为单位缓存数据，没有大字段，缓存数据更多，减少了IO。

垂直分库

垂直分库一般是按照业务对耦合性较小的表进行拆分，比如用户表和订单，进一步可以发展成微服务。

缺点

  1. 需要业务可拆分的前置条件
  2. 业务之间数据关联变得复杂，需要在应用层进行数据连接聚合
  3. 跨库之后分布式事务一致性难以保证

水平分表

单表数据量过大，以mysql为例，一般只能较好地处理千万级的数据，再高则查询效率下降明显。此时考虑对单表水平拆分，即创建一些结构相同的表，将数据分散存储。

表分区

数据库本身提供了类似分表的技术，称为表分区，创建表时定义表为分区表，在物理层将数据拆分成了不同的分区，mysql分区支持以下类型：

  • range
  • list list类型分区与range类似，只是其范围是离散的
  • hash
  • key 前面三种类型要求分区列必须是整型，key只要分区列不是text，blob就行，key类似于hash，但是hash函数是数据库自定义的

水平分库

水平分库的逻辑和分表相同，但是水平分表使用的是同一个数据库实例，不同的表共用IO、CPU资源，性能受到硬件资源限制。水平分库使得扩充了分表各自的硬件资源，但是也带来了分布式的一些问题。

水平分片策略

hash分片

hash分片就是按照表的关键字进行hash运算之后取模，然后按照结果分配到不同的表中。hash分表的数据分布比较均匀，但是扩容升级困难。

范围分片

按照关键字的取值范围进行分片，比如日期，一个月为一个分片。范围分表会有热点问题，即最近的查询都集中在最近一个月的分片中，而历史的分片极少查询。

hash分片与范围分片结合

先进行范围分片，比如以日期分组，今年是一组，明年是一组，在同一组内，按照hash分片，将数据均匀地分散到几台数据库上。这样随着时间推移，数据量增长，只需要添加新的分组，扩容简单，有因为hash没有热点问题。

一致性hash分片

一致性hash类似于范围分片，只不过分片的依据是hash计算结果。

分库分表带来的问题

分库分表会带来很多编程上的复杂性，如果可以，SQL优化、缓存、数据库主从集群应该要优先考虑。

分布式事务一致性问题

主要是分库之后，多数据库无法支持事务一致性。有XA协议和两阶段提交支持跨数据库事务，但是性能不高，一般不采用，而是追求最终一致性。

全局唯一自增主键

需要引入第三方分布式ID组件，提高了复杂度。

数据路由

水平分库分表后，要对数据进行路由，确定数据要落入哪个分区，复杂度提高。

查询问题

在水平分库分表中，我们称作为分库分表基准的列为partition key，如果查询时使用的字段不是partition key，查询无法定位分片。常用的方法有：

  • 映射法：建立映射表，可能作为查询key的列都在这张表中
  • 基因法：partition key的值由可能查询的列作为基因生成

    联合查询

    不管水平还是垂直拆分，由于数据的离散，需要聚合数据都变得更麻烦，一般都需要应用层改造，通过应用层来聚合整理数据。

    分页排序

    这个问题在水平分表分库中比较明显，因为数据分散在不同分片，需要在应用中聚合数据再排序，复杂度提高。

    hash分片扩容问题

    对于水平分库，随着业务进一步增长，势必要考虑数据库扩容。采用常用的hash分片，hash映射结果随着数据库实例变动而变动，这会导致原先落在A库的数据现在要落到B库，对历史数据而言，会出现A库有数据，但是应用按照新的映射会去查B库，导致查不到数据的情况。所以每次扩容要进行数据迁移。

    升级从库法（2主2从->4主4从）

  1. 应用的数据源地址(0/1)由两个变成(0/1/2/3)个，新增的两个地址2指向原来的0，3指向原来的1.
  2. 先修改应用的地址指向，2指向0的备库，3指向1的备库，然后解除现有主从关系，添加新的备库，新建主从关系。
  3. 删除冗余数据

    双写迁移法

  4. 修改应用，除了按旧有规则写入老库外，还按照新规则写入新库
  5. 将老库的数据按照新规则迁移到新库
  6. 修改应用，解除双写
